LINE Delivery Flow

PDF配信の仕組み

LINE Messaging API にはPDF送信タイプがないため、以下の方式で「添付体験に近い」配信を実現する。

Step処理担当
1PDFをSupabase Storageへ保存Worker
2署名付きURL（有効期限付き）を発行Worker
3LINEへ Flex Message で送信
ファイル名 / 会社名 / 人名 / 有効期限 / 「PDFを開く」ボタン		Worker
4受信者がボタンをタップ → /v/:token へアクセス受信者
5トークン検証 → delivery_events に URL_OPENED 記録API
6署名付きURLへリダイレクト → PDFダウンロードブラウザ
